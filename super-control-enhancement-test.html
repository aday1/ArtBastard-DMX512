1. LAMP Control Testing

Test Objectives

Verify that LAMP control functions properly across different fixture types and provides accurate lamp intensity management.

Test Steps

  • Navigate to Super Control panel
  • Select fixtures that have lamp control channels
  • Locate "Lamp Control" slider in Effects section
  • Test slider movement from 0 to 255
  • Verify real-time DMX output changes
  • Check lamp indicator in control indicators panel
  • Test MIDI learn functionality (if MIDI controller available)
  • Verify lamp control works with multiple fixture selection
Expected Behavior:
  • Lamp slider should control lamp-related DMX channels
  • Lamp indicator should show active state when value > 0
  • Real-time feedback in channel monitoring section
  • MIDI learn button should enter learning mode

2. Reset Function Testing

Test Objectives

Validate reset functionality including trigger reset button and manual reset control.

Test Steps

  • Select fixtures with reset/function channels
  • Locate "Reset Function" controls in Effects section
  • Test "Trigger Reset" button
  • Observe auto-reset behavior (value should return to 0 after 500ms)
  • Test manual reset slider for specific values
  • Verify reset indicator in control indicators panel
  • Test reset with different fixture types
  • Confirm reset channels are properly mapped
‚ö†Ô∏è Important: Reset functions may cause fixtures to restart or change operating modes. Be prepared for temporary fixture behavior changes.

3. Direct DMX Channel Controls Testing

Test Objectives

Test individual DMX channel control capabilities and ensure proper channel identification and manipulation.

Test Steps

  • Switch to "Channels" selection mode in Super Control
  • Select multiple DMX channels using TouchDmxControlPanel
  • Verify "Direct DMX Channel Controls" section appears
  • Check channel information display (number, fixture, type)
  • Test vertical sliders for each selected channel
  • Use numerical input fields for precise values
  • Test quick action buttons (0%, 50%, 100%)
  • Verify real-time DMX output changes
  • Test with channels from different fixtures
Visual Elements to Check:
  • Channel numbers clearly displayed
  • Fixture names and channel types shown
  • Vertical sliders respond to mouse/touch input
  • Value displays update in real-time
  • Grid layout adapts to different screen sizes

Test Steps

  • Verify all 8 GOBO images load properly
  • Test clicking each GOBO option
  • Verify active GOBO highlighting
  • Check GOBO values update correctly (0, 32, 64, 96, 128, 160, 192, 224)
  • Test fallback icons if images fail to load
  • Verify traditional GOBO slider still functions
  • Test with disabled state when no fixtures selected

5. Integration and Performance Testing

Overall System Integration

  • Test all new features work together without conflicts
  • Verify performance with multiple features active simultaneously
  • Check responsive design on different screen sizes
  • Test touch interface on mobile/tablet devices
  • Verify all controls work with fixture groups and capabilities
  • Test undo/redo functionality with new controls
  • Verify no console errors or warnings
  • Check memory usage during extended operation

Cross-Browser Compatibility

  • Test in Chrome/Chromium browsers
  • Test in Firefox
  • Test in Safari (if available)
  • Test in Edge browser
  • Verify SVG graphics display correctly across browsers
